Rationale

Au travail, nous déployons actuellement des GKEs pour nos applications, aujourd’hui la containerisation n’est plus quelques chose que je vois comme une option mais une nécessité par rapport à la valeur que celà apporte, bien qu’il faille souvent travailler sur la partie organisation des équipes avant de pouvoir en tirer le meilleur partie, mais ceci fera l’objet d’un autre billet.

Néanmoins, il s’avère que maitriser une tel système n’est pas à la porté de tout le monde , implémenter une gouvernance technique autour de ce ssystème n’est pas chose aisée, et les personnes qui l’implémente sont souvent unanimes, c’est compliqué et il faut un expertise bien réelle pour pouvoir designer, et déployer les différentes stacks avec des choix de technos adaptés au conexte.

J’ai donc très rapidement été curieux et aujourd’hui je tente l’aventure d’un point de vue technique, à savoir l’installation d’un k8s vanilla sur une VM pour démarrer.

Je relate ici les différentes étapes de cette installation et les différents problèmes rencontrés.

Let’s get started

Je me dirige donc vers le site officiel de kubernetes pour commencer et je finis sur la rubrique Learning Environments, mais celà ne m’enchante guère même si c’est une solution simple et rapide pour démarrer. Je vais donc essayer de faire une installation plus “from scratch” plutôt avec les outils utilisés pour déployer des k8s de “prod” et tout commence ici.

Kubeadm

Kubeadm est la toolbox afin de pouvoir démarrer un cluster k8s, un point à noter c’est qu’il est compiler en linkage dynamic et non en static, il faut donc une glibc quelques part (Alpine est sur musl) et donc je vais utiliser une debian pour commencer.

Pre-requisites

Le but n’est pas ici de redire ce qui est déjà dans la doc officielle, mais on s’assurera d’avoir à minima 2 CPUs sur laa VM, un kernel supporté et des MACs uniques pour chaque VM utilisée.

Disable swap

On s’assure aussi de désactiver la swap sur la VM.

1david@node:~$ sudo swapoff -a
2david@node:~$ sudo sed -i '/swap/ s/^/#/' /etc/fstab
3david@node:~$ grep swap /etc/fstab
4## swap was on /dev/sda5 during installation
5#UUID=19d62f10-216b-4978-a8a3-bc8c0a4caa21 none            swap    sw              0       0

Installing runtime

On va installer containerd, c’est le runtime par défaut recommandé pour k8s, comme c’es notre premier déploiement nous allons l’utiliser, Kubernetes utilise la protocole CRI (Container Runtime Interface) pour communiquer avec le runtime, en étant curieux je suis aller voir, il s’agit tout simplement d’une API gRPC avec une sérialisation en protocol buffers, on peut regarder l’implementation proto ici.

Nous allons donc installer containerd via le repository officiel docker, comme indiqu" dans la doc officielle, nous faisons:

 1# Add Docker's official GPG key:
 2sudo apt-get update
 3sudo apt-get install ca-certificates curl
 4sudo install -m 0755 -d /etc/apt/keyrings
 5sudo curl -fsSL https://download.docker.com/linux/debian/gpg -o /etc/apt/keyrings/docker.asc
 6sudo chmod a+r /etc/apt/keyrings/docker.asc
 7
 8# Add the repository to Apt sources:
 9echo \
10  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/debian \
11  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
12  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
13sudo apt-get update

J’installe par la suite uniquement containerd, et pas docker-engine, docker-cli, etc…

1david@node:~$ sudo apt-get install  containerd.io

Inistallation de KubeADM

Nous allons donc installer 3 binaires (from doc):

  • kubelet : the component that runs on all of the machines in your cluster and does things like starting pods and containers.
  • kubeadm: the command to bootstrap the cluster.
  • kubectl : the command line util to talk to your cluster.

INstall avec le repo de kubernetes

On rajoutes les packages nécéssaires (from the doc) :

1sudo apt-get update
2# apt-transport-https may be a dummy package; if so, you can skip that package
3sudo apt-get install -y apt-transport-https ca-certificates curl gpg

Download the public signing key for the Kubernetes package repositories. The same signing key is used for all repositories so you can disregard the version in the URL:

1# If the directory `/etc/apt/keyrings` does not exist, it should be created before the curl command, read the note below.
2# sudo mkdir -p -m 755 /etc/apt/keyrings
3curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.33/deb/Release.key | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg

Add the appropriate Kubernetes apt repository. Please note that this repository have packages only for Kubernetes 1.33; for other Kubernetes minor versions, you need to change the Kubernetes minor version in the URL to match your desired minor version (you should also check that you are reading the documentation for the version of Kubernetes that you plan to install).

1# This overwrites any existing configuration in /etc/apt/sources.list.d/kubernetes.list
2echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.33/deb/ /' | sudo tee /etc/apt/sources.list.d/kubernetes.list

Update the apt package index, install kubelet, kubeadm and kubectl, and pin their version:

1david@node:~$ sudo apt-get update
2sudo apt-get install -y kubelet kubeadm kubectl
3sudo apt-mark hold kubelet kubeadm kubectl
4[...]
5kubelet set on hold.
6kubeadm set on hold.
7kubectl set on hold.
8david@node:~$

Selon la doculentation officielle ici, nouspouvons lancer le kubelet, mais celui-ci va crasher en boucle, c’est pas grave on est là pour faire de la casse !!

1sudo systemctl enable --now kubelet

Nos outils sont maintenant installés et les pré-requis complétés, nous continuons donc avec la création denotre cluster

Création du cluster

Avant la création du cluster il faut d’abord considérer quel plugin CNI nous allons utiliser, en effet, le plugin CNI va avoir un CIDR par défaut qui va être utilisé pour les pods. Ici je vais utiliser le CNI Flannel qui a un CIDR par défaut de 10.244.0.0/16.

On lance la création du cluster :

1david@node:~$ sudo kubeadm --pod-network-cidr=10.244.0.0/16 init
2[init] Using Kubernetes version: v1.33.1
3[preflight] Running pre-flight checks
4[preflight] Pulling images required for setting up a Kubernetes cluster
5[...]

7[...]
